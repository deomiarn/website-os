---
name: gsd:create-roadmap
description: Create WebDesign-OS roadmap with mandatory 8 phases
allowed-tools:
  - Read
  - Write
  - Bash
  - AskUserQuestion
  - Glob
---

<objective>
Create WebDesign-OS project roadmap with the mandatory 8 phases defined in WORKFLOW.md.

This is a LOCAL OVERRIDE for WebDesign-OS projects. It enforces the 8-phase workflow.
</objective>

<context>
@.planning/PROJECT.md
@.planning/config.json
@webdesign-os/WORKFLOW.md
</context>

<process>

<step name="validate">
```bash
# Verify project exists
[ -f .planning/PROJECT.md ] || { echo "ERROR: No PROJECT.md found. Run /gsd:new-project first."; exit 1; }

# Verify WORKFLOW.md exists
[ -f webdesign-os/WORKFLOW.md ] || { echo "ERROR: webdesign-os/WORKFLOW.md not found."; exit 1; }
```
</step>

<step name="check_existing">
Check if roadmap already exists:

```bash
[ -f .planning/ROADMAP.md ] && echo "ROADMAP_EXISTS" || echo "NO_ROADMAP"
```

**If ROADMAP_EXISTS:**
Use AskUserQuestion:
- header: "Roadmap exists"
- question: "A roadmap already exists. What would you like to do?"
- options:
  - "View existing" - Show current roadmap
  - "Replace" - Create new roadmap (will overwrite)
  - "Cancel" - Keep existing roadmap

If "View existing": `cat .planning/ROADMAP.md` and exit
If "Cancel": Exit
If "Replace": Continue
</step>

<step name="read_workflow">
Read the WebDesign-OS workflow definition:

```bash
cat webdesign-os/WORKFLOW.md
```

This file defines the EXACT 8 phases to use. Do NOT derive custom phases.
</step>

<step name="create_structure">
```bash
mkdir -p .planning/phases
mkdir -p .planning/phases/01-content-planning
mkdir -p .planning/phases/02-design-system
mkdir -p .planning/phases/03-shape-pages
mkdir -p .planning/phases/04-write-specs
mkdir -p .planning/phases/05-implement
mkdir -p .planning/phases/06-seo
mkdir -p .planning/phases/07-refine
mkdir -p .planning/phases/08-verify-export
```
</step>

<step name="write_roadmap">
Create `.planning/ROADMAP.md` with EXACTLY these 8 phases from WORKFLOW.md:

```markdown
# WebDesign-OS Roadmap

## Domain Expertise

expertise/webdesign-os (embedded in webdesign-os/WORKFLOW.md)

## Phases

### Phase 1: Content Planning
**Goal:** Define content quantities and structure per page
**Output:** webdesign-os/config/content-inventory.json
**Research:** Unlikely (internal planning)

### Phase 2: Design System
**Goal:** Create visual foundation based on inspirations
**Output:** webdesign-os/config/design-tokens.json
**Research:** Unlikely (uses inspiration images)

### Phase 3: Shape Pages
**Goal:** Design each page section by section
**Output:** webdesign-os/config/page-shapes/*.json
**Research:** Likely (shadcnblocks component selection)

### Phase 4: Write Specs
**Goal:** Create technical implementation specs
**Output:** webdesign-os/specs/*.md
**Research:** Unlikely (internal documentation)

### Phase 5: Implement
**Goal:** Build all pages per specs
**Output:** Next.js project in exports/
**Research:** Likely (Next.js, shadcn/ui setup)

### Phase 6: SEO
**Goal:** Optimize for search engines
**Output:** SEO-optimized pages
**Research:** Unlikely (standard patterns)

### Phase 7: Refine (Optional)
**Goal:** Polish based on feedback
**Output:** Refined implementation
**Research:** Unlikely

### Phase 8: Verify & Export
**Goal:** Final QA and production build
**Output:** Production-ready project
**Research:** Unlikely

## Status

| Phase | Name | Status |
|-------|------|--------|
| 1 | Content Planning | not started |
| 2 | Design System | not started |
| 3 | Shape Pages | not started |
| 4 | Write Specs | not started |
| 5 | Implement | not started |
| 6 | SEO | not started |
| 7 | Refine | not started |
| 8 | Verify & Export | not started |

## Dependencies

1 → 2 → 3 → 4 → 5 → 6 → 7 (optional) → 8

---
*Generated by WebDesign-OS GSD Integration*
```
</step>

</step>

<step name="commit">
```bash
git add .planning/ROADMAP.md .planning/STATE.md .planning/phases/
git commit -m "$(cat <<'EOF'

```
</step>
